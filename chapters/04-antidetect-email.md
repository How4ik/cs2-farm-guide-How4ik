# Антидетект-браузер и почта

Два столпа подготовки аккаунта: **изоляция браузера** (антидетект) и **надёжная почта**. Без них Steam и Outlook либо связывают профили между собой, либо отрезают доступ на полпути.

## Зачем этот этап

| Компонент | Задача |
|-----------|--------|
| Антидетект | Чистый кэш, уникальный отпечаток, один профиль = один «компьютер» |
| Почта | Восстановление Steam, письма Guard, привязка — потеря ящика = потеря аккаунта |
| Разделение Chrome / AdsPower | Почта и Steam не делят один след браузера |

> Почта — **фундамент аккаунта**. Потеряете доступ к ящику — потеряете Steam.

---

## Антидетект-браузер

Антидетект-браузер чистит кэш, **подменяет отпечаток** (fingerprint) браузера и **изолирует профили** друг от друга. Для фермы это обязательный уровень между «домашний Chrome» и «живой отдельный пользователь».

### Рекомендуемая схема

| Инструмент | Для чего | Почему отдельно |
|-----------|---------|-----------------|
| **Chrome (гостевой режим)** | Регистрация и проверка **почты** Outlook | Лёгкая изоляция, без синхронизации Google |
| **AdsPower** | Регистрация и вся работа со **Steam** | Прокси, WebRTC, timezone, Canvas — полный контроль |

### Зачем разделять Chrome и AdsPower

1. **Разные задачи** — почта Microsoft чувствительна к IP при регистрации; Steam — к отпечатку и WebRTC.
2. **Разные сессии** — письмо Guard нельзя открывать в «случайном» Chrome без прокси.
3. **Масштаб** — десятки Steam-профилей в AdsPower; почта — гостевой Chrome + alias.

Смешивание почты и Steam в одном необезличенном браузере создаёт **связи между аккаунтами**, которые Valve и Microsoft могут учитывать.

> AdsPower — не единственный антидетект на рынке (Multilogin, Dolphin и др.), но логика везде похожая: **профиль + прокси + отпечаток**.

### Базовые принципы антидетекта

| Принцип | Действие |
|---------|----------|
| Один профиль — один аккаунт Steam | Не переиспользовать «навсегда» |
| Прокси в профиле = прокси при регистрации | Тот же IP для Guard |
| WebRTC в режиме **«Подмена»** | Иначе реальный локальный IP утекает |
| Timezone и язык = регион прокси | Мелочь в связке с fingerprint |
| Canvas / WebGL — шум по умолчанию | Не отключать всё «для скорости» |

> Не смешивайте аккаунты на одном IP. Не открывайте два фермерских профиля на одном IP «на минутку» — этого достаточно, чтобы связать цепочку.

### Chrome для почты: правила

1. Только **гостевой режим** (Ctrl+Shift+N).
2. **Без** синхронизации Google-аккаунта.
3. Если Chrome уже использовался для регистраций:
   - `Ctrl+Shift+Delete`
   - Период: **«всё время»**
   - Куки, кэш, данные сайтов — очистить
4. VPN/прокси включён **до** открытия outlook.com.

## Почта

### Сравнение провайдеров

| Провайдер | Вердикт | Причина |
|-----------|---------|---------|
| TempMail, TrashMail, FirstMail | **Нельзя** | Valve давно режет одноразовые домены |
| Mail.ru, Яндекс, Rambler | **Лотерея** | Могут сочесть подозрительными |
| **Gmail** | **Можно** | Надёжно, предсказуемо; лимит регистраций на номер |
| **Outlook** | **Рекомендуется** | Без телефона, alias на одну учётку, масштаб |

### Gmail vs Outlook для фермы

| Критерий | Gmail | Outlook |
|----------|-------|---------|
| Надёжность для Steam | Высокая | Высокая |
| Телефон при регистрации | Часто требуется | Не обязателен |
| Масштаб (много ящиков) | Лимит на номер | **Alias** — много «почт», один inbox |
| Steam и alias | Отдельные адреса | Каждый alias = отдельная почта для Steam |
| Рекомендация гайда | Если мало аккаунтов | **Основной выбор для масштаба** |

**Outlook удобнее для масштаба:** Steam считает каждый **alias** отдельной почтой, все письма приходят в **один входящий ящик**. Проверять десять регистраций — один логин.

> Outlook и Gmail — скучно, зато предсказуемо. Alias на Outlook особенно удобен, когда нужно десять почт, а проверять хочется один inbox.

### Правила имён и логинов

**Не использовать** шаблонные имена:

- `FarmAcc01`
- `CS2Bot_3`
- `DropFarmer99`
- `farm_mail_17`

За похожие логины в Steam часто банят **цепочкой** — система видит паттерн фермы.

**Использовать:**

- Случайное словосочетание из генератора (например: `quiet-river-84`, `marble-cloud-k7`)
- Без общего префикса на всех аккаунтах
- Уникальное имя профиля AdsPower (`india_042`, не `cs2farm99`)

### Хранение паролей и привязок

| Данные | Как хранить |
|--------|-------------|
| Пароль Outlook | В таблице, отдельная колонка |
| Пароль Steam | Отдельно от Outlook, в той же таблице |
| Alias → аккаунт Steam | Явная пометка в таблице |
| Прокси для ящика | Тот же host:port, что у Steam-профиля |

> Через месяц без записей вы сами себе враг: забудете, какой alias к какому аккаунту привязан.

## Пошаговая связка (обзор)

1. Включить прокси на регион (Индия).
2. Chrome гостевой → регистрация Outlook → alias.
3. AdsPower: тот же прокси → профиль → Steam на том же IP.
4. Письма Steam и Guard — только через прокси (AdsPower или тот же IP).

## Частые ошибки

| Ошибка | Последствие |
|--------|-------------|
| Одноразовая почта | Steam не принимает / блок позже |
| Один Chrome с куками на все регистрации | Связанные следы |
| Шаблонные логины `farm_*` | Цепочный бан |
| Почта и Steam в одном «обычном» браузере | Перекрёстные fingerprint |
| WebRTC не в режиме подмены | Утечка реального IP в AdsPower |
| Логин в Outlook с другого континента сразу после регистрации | Ящик «умирает» — дайте отлежаться 2+ часа на том же IP |
| Один alias на два Steam-аккаунта | Путаница и риск при восстановлении |
