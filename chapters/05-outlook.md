# Регистрация Outlook

Первый практический шаг в цепочке фермы — **регистрация почты Outlook**. От качества ящика зависит регистрация Steam, привязка Guard и возможность восстановить доступ. Ниже — полная пошаговая инструкция «как будто сидим рядом за одним монитором».

## Зачем этот этап

| Причина | Пояснение |
|---------|-----------|
| Фундамент аккаунта | Steam привязан к почте; без ящика — нет контроля |
| Масштаб через alias | Один Outlook → много адресов для Steam, один inbox |
| Предсказуемость | Valve режет одноразовые домены; Outlook проходит стабильно |
| Согласованность с регионом | Страна в форме + IP = Индия → дешёвый Prime и корректные gift-карты |

> Почта готова — только после этого переходим к Steam через AdsPower. Порядок нарушать нельзя.

---

## Подготовка

### Шаг 1 — Прокси на регион

1. Включите **VPN или прокси** на выбранный регион (в гайде — **Индия** под дешёвый Prime).
2. Убедитесь, что подключение **стабильно** — не переключайте страну посреди регистрации.

### Шаг 2 — Chrome в гостевом режиме

1. Откройте **Chrome строго в гостевом режиме** (Ctrl+Shift+N).
2. **Не** входите в Google-аккаунт, **не** включайте синхронизацию.
3. Гостевой режим — чтобы куки и кэш не тянули следы прошлых регистраций.

### Шаг 3 — Очистка (если Chrome уже использовался)

Если этот Chrome когда-либо использовался для других регистраций:

1. Нажмите **Ctrl+Shift+Delete**.
2. Период: **«всё время»**.
3. Отметьте: **куки**, **кэш**, **данные сайтов**.
4. Подтвердите очистку.
5. Закройте обычные окна, снова откройте **только гостевой** режим.

## Проверка IP (обязательно)

Перед переходом на outlook.com проверьте IP на любом сервисе (2ip.ru, whoer.net и т.п.).

| Что проверить | Ожидание |
|---------------|----------|
| Страна | Совпадает с VPN (например, **Индия**) |
| Город / регион | В пределах выбранной страны, не «ваш» домашний город в другой стране |
| Утечки | Нет явного несоответствия «VPN Индия, DNS Европа» |

> Если вместо Индии вылезает Европа или ваш реальный регион — Outlook запомнит несостыковку, и ящик может отвалиться уже на этапе Steam.

**Если гео не совпало:** смените прокси, переподключите VPN, начните подготовку заново. Не регистрируйтесь на «почти правильном» IP.

## Пошаговая инструкция: регистрация

### 1. Открыть Outlook

1. В гостевом Chrome перейдите на [outlook.com](https://outlook.com).
2. Нажмите **«Создать учётную запись»** (Create free account).

### 2. Адрес электронной почты

1. Придумайте адрес **без шаблонов**:
   - **Плохо:** `farm_mail_17`, `cs2_acc_04`, `dropfarmer99`
   - **Хорошо:** случайное словосочетание из генератора
2. Запомните или сразу запишите адрес в таблицу аккаунтов.

### 3. Пароль

1. Задайте **надёжный уникальный** пароль.
2. Сохраните в таблице — **отдельно** от будущего пароля Steam.

### 4. Имя и фамилия

1. Укажите правдоподобные имя и фамилию (латиница).
2. Избегайте одинаковых «Иван Иванов» на всех аккаунтах подряд.

### 5. Страна и дата рождения

| Поле | Правило |
|------|---------|
| **Страна** | **Должна совпасть с VPN** (Индия → India) |
| **Дата рождения** | **Не позже 2004 года** — система должна видеть взрослого пользователя |

### 6. Капча

1. Пройдите проверку, если запросят.
2. Если капча отвечает **«Что-то пошло не так»** — см. раздел ниже.

### 7. Завершение

1. Дождитесь создания ящика.
2. Войдите в почту, убедитесь, что inbox открывается.
3. **Не** переключайте IP минимум **2–3 часа** — дайте ящику «отлежаться» на том же адресе.

---

## Если капча: «Что-то пошло не так»

Это почти всегда **грязный IP**, а не ваша вина.

| Действие | Да |
|----------|-----|
| Сменить прокси / переподключить VPN | Да |
| Очистить гостевой Chrome и начать заново | Да |
| Долбить ту же связку 10 раз подряд | **Нет** |

Алгоритм: новый IP → проверка гео → новая попытка с **новым** именем ящика.

## Пошаговая инструкция: создание alias

Alias нужен, чтобы для Steam использовать **отдельный адрес**, а проверять письма в **одном** ящике.

1. Войдите в Outlook **с того же IP**, что при регистрации.
2. Откройте **«Сведения»** (Account info / Параметры учётной записи).
3. Перейдите **«Изменить данные учётной записи»**.
4. Выберите **«Добавить адрес электронной почты»**.
5. Создайте **alias** — новый адрес на ту же учётку.
6. Запишите в таблицу: `основной ящик` → `alias для Steam #N`.

| Для Steam | Для вас |
|-----------|---------|
| Alias = **отдельная почта** | Все письма в **один входящий** |

> Пароль от Outlook храните отдельно от пароля Steam, но в **одной таблице** с пометкой, какой alias к какому аккаунту привязан.

## Хранение данных

| Что | Где |
|-----|-----|
| Основной адрес Outlook | Таблица, колонка «почта» |
| Alias для Steam | Таблица, колонка «alias» |
| Пароль Outlook | Таблица, защищённый файл |
| Прокси при регистрации | `host:port:login:password` — тот же для Steam |
| Дата регистрации ящика | Для контроля «отлежки» |

## Чек-лист

- [ ] Прокси включён, IP = выбранный регион (Индия)
- [ ] Chrome только гостевой, без Google-синхронизации
- [ ] Кэш/куки очищены, если браузер не «чистый»
- [ ] Страна в форме = страна по IP
- [ ] Дата рождения ≤ 2004
- [ ] Логин без шаблонов `farm_*`, `cs2bot_*`
- [ ] Регистрация успешна, inbox открывается
- [ ] Alias создан и записан в таблицу
- [ ] 2+ часа на том же IP перед активной сменой гео
- [ ] Прокси тот же пойдёт в AdsPower для Steam

## Частые ошибки

| Ошибка | Последствие |
|--------|-------------|
| Регистрация без проверки IP | Ящик отвалится на Steam |
| Обычный Chrome с куками | Связь с другими регистрациями |
| Страна в форме ≠ VPN | Несостыковка, блокировки |
| Сразу логин с другого континента | Outlook «умирает» |
| Шаблонное имя `farm_mail_17` | Риск цепочного бана в Steam |
| Нет alias — основной ящик на всё | Сложнее масштаб; путаница |
| Пароли Outlook и Steam одинаковые | Компрометация обоих сразу |
| 10 попыток капчи на одном грязном IP | Потеря времени, метка на IP |
| Письмо Steam позже открыли без прокси | Новый след, проблемы с Guard |
