# Схема дропа через буферный аккаунт

## Зачем

Кроме **хранилки** нужны **буферные аккаунты** с открытым трейдом. Valve и внутренние системы смотрят на **массовые переводы** на один кошелёк — десятки одинаковых кейсов с фермы напрямую на хранилку выглядят как один «луч» в одну точку.

| Без буфера | С буфером |
|------------|-----------|
| 50 ферм → 1 хранилка | 50 ферм → 5–10 буферов → хранилка |
| Один trace на дорогой инвентарь | Распределённый поток |
| Красная табличка при аномалии | Меньше риска агрегации |

> ❗ **Важно**
>
> **Никогда** не кидайте дроп на **основной аккаунт** с дорогим инвентарём. Буферный аккаунт — не paranoia, а **гигиена**.

### Роли в цепочке

| Звено | Назначение | Prime | Фарм |
|-------|------------|-------|------|
| **Ферма** | Еженедельный дроп | ✅ Да | ✅ Да |
| **Буфер** | Промежуточный транзит | ❌ Нет | ❌ Нет |
| **Хранилка** | Долгосрочное хранение ценности | ❌ Не нужен | ❌ Нет |

### Схема потока

```
Фермерский аккаунт (дроп) → Буфер (транзит) → Хранилка / Продажа / TF2-ключи
```

## Пошагово

### Шаг 1 — Создание буферного аккаунта

Регистрируйте буфер по **той же дисциплине**, что и ферму:

1. **Outlook** с alias — отдельная почта на буфер
2. **AdsPower** — отдельный профиль, отдельный прокси (не пул фермы!)
3. **Steam** — регистрация под регионом прокси
4. **NebulaAuth** — Guard с **того же IP**, maFile + R-код в таблицу
5. **Gift card $5** — открытие маркета (без Prime!)

| Параметр буфера | Значение |
|-----------------|----------|
| Prime | **Не покупать** |
| Фарм CS2 | **Не вести** |
| Маркет | **Открыт** ($5) |
| Связь с основным профилем | **Нет** (не в друзья, не один IP) |
| Прокси | **Свой**, не хранилка и не ферма |

> ⚠️ **Внимание**
>
> Не покупайте Prime на буфере «для красоты» — лишняя активность на транзитной точке.

### Шаг 2 — Соотношение буферов и ферм

| Правило | Значение |
|---------|----------|
| Буферов на ферму | **1 буфер на 5–10 ферм** |
| Не делать | 1 буфер на 100 ферм |
| Хранилок | 1–2 на всю систему, тихие, редкие входы |

Пример на 30 ферм:

| Буфер | Фермы | Хранилка |
|-------|-------|----------|
| buffer_01 | farm_01 … farm_10 | — |
| buffer_02 | farm_11 … farm_20 | storage_main |
| buffer_03 | farm_21 … farm_30 | — |

### Шаг 3 — Еженедельный дроп на ферме

1. Фермерский аккаунт получает **еженедельный дроп** (кейс / скин)
2. Предмет остаётся на ферме до **месячного сбора**
3. Запишите в таблицу: аккаунт, предмет, дата дропа, привязанный буфер
4. **Не отправляйте** сразу на хранилку

### Шаг 4 — Месячный сбор на буфер

**Раз в месяц, не чаще:**

1. Соберите дроп с пачки ферм (5–10 штук) на **один буфер**
2. Подтвердите трейды через NebulaAuth (прокси буфера и фермы — свои)
3. **Не делайте** 50 трейдов в одну минуту — разнесите на часы / день
4. Ждите окончания **trade hold** (7 дней), если предметы с маркета или обмена с hold

| Частота | Можно | Нельзя |
|---------|-------|--------|
| Сбор на буфер | 1 раз / месяц | Каждую неделю с 20 ферм |
| Трейды | Несколько в день | 30 одновременно |

### Шаг 5 — Перевод с буфера на хранилку

1. После hold (если был) переводите с буфера на **хранилку**
2. **Пакетом, но без фанатизма:**

| Подход | Риск |
|--------|------|
| Несколько **мелких** трейдов в **разные дни** | Ниже |
| Один жирный перевод на **$500** | Выше |

3. Хранилка:
   - **Не** тот же прокси-пул, что регистрация фермы
   - **Не** в друзьях у всех ферм
   - **Не** Prime «для красоты»
   - Редкие входы с **отдельного IP**

### Шаг 6 — Альтернативный маршрут через TF2-ключи

Ещё **осторожнее** вариант — меньше «одинаковых кейсов подряд»:

1. На **маркете буфера** продайте часть дропа за кошелёк Steam
2. Купите **Mann Co. Supply Crate Key** (TF2)
3. Перекиньте **ключи** на хранилку или внешнюю площадку
4. Ключи **ликвиднее** и меньше бросаются в глаза, чем десять одинаковых кейсов

| Маршрут | Скорость | Потери | Заметность |
|---------|----------|--------|------------|
| Кейсы напрямую | Быстро | Меньше комиссий на маркете | Выше |
| TF2-ключи | Средне | Комиссии маркета + спред ключей | Ниже |

### Шаг 7 — Таблица учёта

Добавьте колонки:

| Колонка | Пример |
|---------|--------|
| Ферма | india_042 |
| Буфер | buffer_01 |
| Хранилка | storage_main |
| Дата дропа | 2026-03-10 |
| Дата сбора на буфер | 2026-04-01 |
| Предмет | Kilowatt Case |
| Hold до | 2026-04-08 |
| Дата на хранилку | 2026-04-10 |
| Метод вывода | ключи / TradeON |

## Чек-лист

### Создание буфера

- [ ] Отдельная почта (Outlook + alias)
- [ ] Отдельный прокси (не ферма, не хранилка)
- [ ] AdsPower-профиль изолирован
- [ ] NebulaAuth + maFile + R-код
- [ ] $5 gift, маркет открыт
- [ ] **Prime не куплен**
- [ ] Нет связи с основным Steam (друзья, IP)

### Эксплуатация

- [ ] Соотношение 1 буфер : 5–10 ферм
- [ ] Сбор на буфер **не чаще 1 раза / месяц**
- [ ] Трейды разнесены по времени
- [ ] Hold соблюдён перед следующим шагом
- [ ] На хранилку — мелкие трейды в разные дни

### Безопасность

- [ ] Не заходить на буфер с домашнего IP
- [ ] Нет личных данных на буфере
- [ ] Хранилка на отдельном прокси-пуле
- [ ] Дроп **не** на основной аккаунт

### Альтернатива

- [ ] При необходимости — продажа на маркете буфера → TF2-ключи → хранилка

## Ошибки

| Ошибка | Последствие | Что делать |
|--------|-------------|------------|
| Дроп напрямую на хранилку | Массовый луч на дорогой инвентарь | Всегда через буфер |
| 1 буфер на 100 ферм | Аномальный объём на одном аккаунте | 1:5–10 |
| Сбор каждую неделю с 20 ферм | Подозрительная частота | Раз в месяц |
| Игнор trade hold | Отмена / флаги | Ждать 7 дней |
| Один трейд $500 на хранилку | Красная табличка | Дробить по дням |
| Prime на буфере | Лишняя активность | Только $5 маркет |
| Буфер с IP фермы | Связка звеньев | Отдельный прокси |
| Вход на буфер с домашнего Wi-Fi | Новый след | Только прокси буфера |
| 10 одинаковых кейсов подряд | Заметный паттерн | TF2-ключи или разнесение |
| Хранилка в друзьях у всех ферм | Граф связей | Максимальная тишина |
| Нет записей в таблице | Путаница, ошибочный трейд | Колонка «буфер» обязательна |

> ⚠️ **Внимание**
>
> Буфер без **hold-дисциплины** — красная табличка. Не кладите на буфер личные данные и не заходите с домашнего IP.

> ℹ️ **Информация**
>
> Красные таблички на такой схеме **почти не видел**, но «почти» — не «никогда». Дисциплина важнее схемы.

> ❗ **Важно**
>
> Буфер добавляет **ещё один hold** в цепочке — зато спите спокойнее. Заложите это в юнит-экономику аккаунта.
