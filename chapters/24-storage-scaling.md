# Хранилка и масштабирование

Аккаунт, куда в итоге **стекается ценность** — но он не должен **светиться** в той же цепочке, что ферма. Хранилка — тихий сейф. Ферма — конвейер.

---

## Зачем отдельная хранилка

Valve смотрит на **массовые переводы на один кошелёк**. Если 50 фермерских аккаунтов шлют дроп на профиль с дорогим инвентарём и друзьями — это палевный луч в одну точку. Хранилка + буфер разрывают связь: ферма не знает, где «дом» ценности.

> Никогда не кидайте дроп на **основной** аккаунт с дорогим инвентарём. Буферный аккаунт — не paranoia, а **гигиена**.

---

## Хранилка: правила

| Правило | Почему |
|---------|--------|
| **Не тот же прокси-пул**, что для регистрации фермы | Связь по IP-пулу |
| **Не добавлять в друзья** все фермерские профили разом | Граф связей |
| **Не покупать Prime** «для красоты» | Лишняя активность, не нужна |
| **Guard + маркет** при необходимости | Минимум функций |
| **Редкие входы** с **отдельного** IP | Тихий аккаунт |
| **Отдельная почта и maFile** | Изоляция от фермы |
| **Не фармить** на хранилке | Только приём ценности |

Хранилка — **максимально тихая**: Guard, маркет при необходимости, редкие входы с отдельного IP.

### Пошагово: создать хранилку

1. Зарегистрируйте Outlook + Steam по той же дисциплине (прокси, AdsPower)
2. Используйте **другой** прокси-пул — не тот, что для фермы
3. Привяжите Guard через NebulaAuth
4. Пополните на $5 для маркета (gift под регион)
5. **Не** покупайте Prime, **не** фармите, **не** добавляйте ферм в друзья
6. Запишите в таблице на листе «Хранилки» — минимум полей
7. Вход раз в 2–4 недели — только для приёма с буфера

### Чек-лист хранилки

- [ ] Отдельный прокси (не фермерский пул)
- [ ] Prime **не** куплен
- [ ] Нет друзей из фермерских аккаунтов
- [ ] Guard работает, maFile в бэкапе
- [ ] Маркет открыт (если нужен для приёма/продажи)
- [ ] В таблице на отдельном листе

## Буфер: мост между фермой и хранилкой

| Параметр | Значение |
|----------|----------|
| Роль | Транзит, не накопление |
| Prime | **Не нужен** |
| Фарм | **Нет** |
| Маркет | **Да**, открытый |
| Соотношение | 1 буфер на **5–10** ферм |
| Частота сбора | **Раз в месяц**, не чаще |
| Hold | Ждать 7 дней если предметы с маркета |

### Пошагово: настроить буфер

1. Регистрация как ферма (Outlook, Steam, Guard, $5 gift)
2. **Без** Prime и без фарма
3. Отдельный прокси, отдельная группа в NebulaAuth (`buffers_march`)
4. В таблице ферм — колонка «Буфер»: `buffer_01`
5. Раз в месяц: дроп с 5–10 ферм → buffer_01
6. Hold → продажа / ключи / TradeON → хранилка **мелкими** трейдами

## Схема аккаунтов

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Ферма 1   │     │   Ферма 2   │     │   Ферма N   │
│  (Prime)    │     │  (Prime)    │     │  (Prime)    │
└──────┬──────┘     └──────┬──────┘     └──────┬──────┘
       │                   │                   │
       └───────────────────┼───────────────────┘
                           ▼
                    ┌─────────────┐
                    │   Буфер     │
                    │ (транзит)   │
                    └──────┬──────┘
                           ▼
                    ┌─────────────┐
                    │  Хранилка   │
                    │  (тихая)    │
                    └──────┬──────┘
                           ▼
                    ┌─────────────┐
                    │   Вывод     │
                    │ Ключи /     │
                    │ TradeON     │
                    └─────────────┘
```

**Поток ценности:** ферма (еженедельный дроп) → буфер (ежемесячный сбор) → хранилка (редко) → реал (ключи / TradeON).

## Изоляция личного и фермерского

> **Не смешивайте** личную игру и ферму на одном железе без изоляции.

| | Личный Steam | Ферма |
|---|-------------|-------|
| Где | Вне песочницы, домашний IP | AdsPower / Sandbox, прокси |
| Друзья | Да | Нет связи с личным |
| Инвентарь | Дорогие скины | Только дроп |
| Prime | Для игры | Для фарма |
| Guard | Телефон / SDA личный | NebulaAuth, отдельные maFiles |

Ферма — **отдельная вселенная**.

Одна случайная авторизация «с основного» в фермерском клиенте может связать то, что вы **месяцами** разводили.

### Чек-лист изоляции

- [ ] Личный Steam не установлен в песочнице Avast
- [ ] Нет общих прокси между личным и фермой
- [ ] Разные почты, разные Guard
- [ ] На личном аккаунте нет друзей из фермы

## Масштабирование: с 3 аккаунтов до фермы

### Зачем начинать с трёх

> Про масштаб: не разгоняйтесь с десяти до ста аккаунтов за неделю. Лучше **месяц на отладку** процесса на малой пачке, потом клонировать схему.

Три аккаунта позволяют:

- Поймать ошибку регистрации **до** 30 Prime
- Пройти **полный цикл** до вывода одного кейса через буфер
- Понять реальные сроки hold, маркета, дропа
- Оценить железо: тянет ли 3 → 7 клиентов

### Рекомендуемый путь

| Этап | Действие | Аккаунтов | Срок |
|------|----------|-----------|------|
| **0** | Прочитать гайд, купить прокси, таблица | 0 | 1–2 дня |
| **1** | Пройти весь цикл вручную, **запись экрана** | **1** | 6–8 недель |
| **2** | Отладить процесс, первый вывод через буфер | **3** | +2–4 недели |
| **3** | Первая рабочая пачка | **10** | +1 месяц |
| **4** | Масштабирование пачками | **10 → 30 → 100** | +1 мес. на удвоение |
| **5** | Автоматизация (панели) | По бюджету | После стабильного 10 |

### Пошагово: этап 1 (один аккаунт)

1. День 1: Outlook + Steam — записать экран
2. День 2: Guard — записать экран
3. День 3: Gift $5 — проверить маркет
4. Недели 2–3: прогрев или skip — зафиксировать решение
5. Prime + Retake lvl 2
6. Sandbox + SRT + первый дроп
7. Буфер → вывод (ключи для скорости первого раза)
8. Заполнить таблицу **полностью** — это шаблон для пачки

### Пошагово: этап 2 (три аккаунта)

1. Клонировать процесс этапа 1 на акк. 2 и 3
2. **10–15 аккаунтов в день** не нужно — 1–2/день достаточно
3. Один буфер на все три
4. Одна хранилка (или вывод с буфера напрямую на первом цикле)
5. Первый **месячный** сбор дропа
6. Посчитать ROI: Prime + прокси vs выведено

### Пошагово: этап 3 (десять аккаунтов)

1. Добавлять по **2–3 аккаунта в день**, не 10 в понедельник
2. Два буфера по 5 ферм
3. Retake / Sandbox: пачка 7 + 3 — или два захода
4. Панель — если ручной фарм не тянет железо
5. Еженедельная рутина из главы «Операционка»

## Юнит-экономика масштаба

| Масштаб | Prime (разово) | Прокси/мес. | Дроп/мес. (грубо) | Рутина |
|---------|----------------|-------------|-------------------|--------|
| 3 акк. | ~$35 | ~$6 | ~12 кейсов | 1 ч/нед. |
| 10 акк. | ~$115 | ~$20 | ~40 кейсов | 3 ч/нед. |
| 30 акк. | ~$345 | ~$60 | ~120 кейсов | 8 ч/нед. + панель |
| 100 акк. | ~$1150 | ~$200 | ~400 кейсов | Панель + 2 ПК |

> Считайте **медиану** кейса, не нож. Редкие дропы — бонус.

## Дроп и кейсы: ожидания

| Факт | Действие |
|------|----------|
| 1 аккаунт = 1 кейс/нед. | Умножайте на кол-во, не на мечты |
| Редкий нож | Не в бизнес-плане |
| Условия Valve | Prime + lvl 2 + активность |
| После патча | Проверить чаты, не паниковать |

## Чек-лист: готовность к масштабу

### Хранилка и буфер

- [ ] Хранилка создана: тихая, без Prime, отдельный IP
- [ ] Буфер(ы): 1 на 5–10 ферм, маркет открыт
- [ ] Схема ферма → буфер → хранилка понятна и записана

### Три аккаунта

- [ ] 3 аккаунта прошли полный цикл
- [ ] Хотя бы 1 кейс выведен через буфер в реал
- [ ] Таблица заполнена, рутина работает

### Изоляция

- [ ] Личный Steam отделён от фермы
- [ ] Нет смешения прокси, Guard, друзей

### Масштаб

- [ ] Этап 1 (1 акк.) записан на видео / в заметках
- [ ] Темп: не 10→100 за неделю
- [ ] Железо / панель готовы к следующей пачке
- [ ] ROI посчитан — понимаете, когда окупается Prime
