# Выбор серверов: Simple Runtime Tracker (SRT)

## Зачем

**Simple Runtime Tracker (SRT)** — инструмент, который отделяет «стою в очереди 20 минут» от «фармлю дроп по кругу». Без учёта **часового пояса** и **нагрузки серверов** вы тратите электричество и время впустую.

| Без SRT | С SRT |
|---------|-------|
| Случайный сервер, пик онлайна | Целевой регион + ночной слот |
| Очередь 15–25 мин | Матч 2v2 за 2–5 мин |
| Больше репортов от живых игроков | Меньше конфликтов |
| Клиент открыт дольше → нагрузка на железо | Короткие сессии, меньше износ |

### Почему 2v2, а не 5v5

Сейчас для дропа упор на **2v2**:

| Режим | Репорты | Длина раунда | Очередь ночью |
|-------|---------|--------------|---------------|
| **2v2** | Меньше | Короче | Быстрее |
| **5v5** | Чаще | Дольше | Длиннее |

5v5 чаще приносит репорты и дольше длится — для фермы это лишний риск и нагрузка.

### Роль часового пояса

Valve распределяет серверы по регионам. **Онлайн** в Индии в 15:00 и в 03:00 — разный. SRT + [24timezones.com](https://24timezones.com) помогают попасть в окно, когда:

- Очередь короче
- Меньше живых игроков мешают
- Матчи 2v2 собираются быстрее

## Пошагово

### Шаг 1 — Установка и первый запуск SRT

1. Скачайте **Simple Runtime Tracker** из проверенного источника (ссылка в описании ролика)
2. Установите на машину, где крутится фарм (или отдельно для анализа)
3. Запустите от имени администратора при необходимости
4. Убедитесь, что SRT видит сеть и может опрашивать серверы CS2

### Шаг 2 — Определение целевого региона

1. Откройте [24timezones.com](https://24timezones.com)
2. Выберите регион аккаунта / прокси:

| Регион аккаунта | Часовой пояс для расчёта |
|-----------------|--------------------------|
| Индия | IST (UTC+5:30) |
| Юго-Восточная Азия (SEA) | Зависит от страны (UTC+7 … +8) |
| Другой | Сверить с IP прокси |

3. Запишите **локальное время**, когда у вас сейчас ночь в целевом регионе

### Шаг 3 — Выбор ночного слота

Оптимальное окно по опыту из гайда:

| Параметр | Значение |
|----------|----------|
| Локальное время региона | **01:00 – 12:00 ночи** |
| Пик избегать | 12:00 – 23:00 днём |
| Сдвиг при росте очереди | **−1 час** раньше, не смена софта |

**Почему ночь:**

| Время | Онлайн | Очередь | Репорты |
|-------|--------|---------|---------|
| **01:00–12:00** (лок.) | Ниже | 2v2 быстрее | Меньше |
| **День** | Выше | Длиннее | Больше живых игроков |

### Шаг 4 — Работа со списком серверов в SRT

1. Откройте SRT → список серверов CS2 для вашего региона
2. Отсортируйте по:
   - Задержке (ping)
   - Текущей нагрузке (если отображается)
   - Стабильности (ваши заметки за неделю)
3. Выберите **3–5 серверов** для ротации — не один IP на все аккаунты
4. Запишите в таблицу:

| Регион | Локальное время старта | Сервер / хост | Средняя очередь |
|--------|------------------------|---------------|-----------------|
| Индия | 01:00–04:00 | xxx.xxx.xxx | 3 мин |
| SEA | 02:00–05:00 | yyy.yyy.yyy | 5 мин |

### Шаг 5 — Распределение пачки аккаунтов

1. **Не гоняйте** все аккаунты на **один IP сервера** в одну секунду
2. SRT даёт список — **распределяйте** пачки по разным хостам внутри региона
3. Пример на 7 аккаунтов:

| Аккаунт | Песочница | Сервер (из SRT) | Сдвиг старта |
|---------|-----------|-----------------|--------------|
| 1–2 | SB-1, SB-2 | Сервер A | 00:00 |
| 3–4 | SB-3, SB-4 | Сервер B | +2 мин |
| 5–7 | SB-5…7 | Сервер C | +5 мин |

4. Режим поиска: **2v2 Wingman** (или актуальный 2v2 в патче)

### Шаг 6 — Еженедельная калибровка

1. Раз в неделю обновляйте таблицу слотов
2. Если после **обновления CS2** очередь выросла:
   - **Сдвиньте таймслот на час раньше** по местному времени
   - **Не меняйте** сразу весь софт / панель
3. Через неделю у вас будет **свой** оптимальный слот, а не чужой совет из чата

### Шаг 7 — Интеграция с ручным фармом и панелями

| Способ фарма | Как использовать SRT |
|--------------|---------------------|
| **Ручной (Sandbox)** | Выбираете сервер вручную по SRT перед запуском каждой пачки |
| **Панель** | Задаёте регион / приоритет серверов в настройках панели по данным SRT |
| **Retake (lvl)** | Тот же регион поиска для всей пачки |

## Чек-лист

### Настройка

- [ ] SRT установлен и запускается
- [ ] 24timezones.com — целевой регион определён
- [ ] Ночной слот **01:00–12:00** локального времени выбран
- [ ] Таблица слотов создана

### Перед каждой сессией фарма

- [ ] Проверено локальное время региона (ночь)
- [ ] Список серверов в SRT обновлён
- [ ] 3–5 серверов для ротации выбраны
- [ ] Режим **2v2**, не 5v5
- [ ] Аккаунты распределены по разным хостам

### Еженедельно

- [ ] Средняя очередь записана в таблицу
- [ ] При росте очереди — сдвиг −1 час
- [ ] После патча CS2 — перепроверка, не паника-смена софта

## Ошибки

| Ошибка | Последствие | Что делать |
|--------|-------------|------------|
| SRT без учёта часового пояса | Фарм днём в пик | 24timezones + ночной слот |
| Все 7 аккаунтов на один сервер | Очередь, репорты, один trace | Ротация по хостам |
| 5v5 днём в пик | Репорты, долгие матчи | 2v2 ночью |
| Смена панели после каждого патча | Хаос, потеря настроек | Сначала −1 час слота |
| Игнор роста очереди после апдейта | Часы впустую | Сдвиг таймслота, не софта |
| Один слот «из чата» без замеров | У вас другой прокси / регион | Своя таблица за неделю |
| Фарм в неправильном регионе | Долгая очередь, высокий ping | Регион = регион прокси |
| Дневной фарм «потому что удобно» | Лишнее электричество и риск | Дисциплина слота |
| SRT установлен, но сервер не меняют | Привыкли к первому хосту | Ротация 3–5 серверов |
| Панель без 2v2 сценария | 5v5 по умолчанию | Обновить сценарий / другая панель |

> ⚠️ **Внимание**
>
> SRT без учёта часового пояса — **пустая трата электричества**. Инструмент работает только в связке с ночным окном региона.

> ❗ **Важно**
>
> Панель не спасёт от бана, если вы фармите **5v5 днём в пик онлайна**. SRT — обязательный компаньон и для ручного фарма, и для панелей.
