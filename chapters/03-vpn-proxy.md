# VPN и прокси

Перед первой регистрацией нужен **платный VPN или личный прокси**. Это не опция «для галочки» — без изоляции IP ферма связывается в одну цепочку, которую Steam и Outlook видят как подозрительную активность.

## Зачем этот этап

Прокси решает три задачи сразу:

1. **Регион** — цена Prime, валюта магазина, gift-карты под страну.
2. **Изоляция** — один аккаунт = один IP; ферма не пересекается с домашним адресом.
3. **Согласованность** — Outlook, Steam, Guard и письма подтверждения должны видеть **один и тот же** географический след.

> Прокси-утилиты позволяют стабильно держать один IP на один профиль и не смешивать следы. Если планируете хотя бы **10 аккаунтов**, закладывайте бюджет на прокси **сразу**.

---

## Почему не бесплатный VPN

| Проблема бесплатных сервисов | Последствие для фермы |
|------------------------------|----------------------|
| Один IP на тысячи пользователей | Steam видит «толпу» на одном адресе |
| IP уже в чёрных списках | Капча Outlook, блокировки Steam |
| Нестабильный гео | Страна в форме ≠ страна по IP → ящик отвалится |
| Нет привязки IP к профилю | Невозможно повторить тот же след для Guard |

> Регистрировать на бесплатном адресе ферму, а потом закидывать туда деньги — плохая идея: аккаунты связаны одним следом, риск ограничений и банов растёт.

## Что такое «хороший» VPN / прокси

**Хороший** — тот, где у вас **выделенный IP или личный прокси**, которым не пользуются посторонние.

| Признак качества | Описание |
|-----------------|----------|
| Выделенный IP | Только ваш трафик на этом адресе |
| Стабильный гео | Страна и город не «прыгают» между сессиями |
| Формат для софта | `host:port:login:password` для AdsPower и NebulaAuth |
| Тип | `socks5` или `http` — по документации провайдера |

### Рекомендуемая связка

| Инструмент | Назначение |
|-----------|-----------|
| **Happ Proxy Utility** | Управление и подключение прокси на системе |
| **Node Proxy** | Прокси-сервер / панель провайдера |

> Это не рекламный блок — просто связка, которая работает на практике. Ссылки на инструменты — в [Telegram @How4ik_Prime](https://t.me/How4ik_Prime).

### Пошаговая настройка (общая логика)

1. Оформите подписку у провайдера с IP под нужный регион.
2. Установите **Happ Proxy Utility** и настройте подключение по инструкции провайдера.
3. При необходимости используйте **Node Proxy** как панель выдачи серверов.
4. Перед каждой регистрацией проверяйте IP на сервисе вроде 2ip.ru или аналоге.
5. Скопируйте данные прокси в AdsPower и NebulaAuth **без лишних пробелов**.

## Базовые правила

| № | Правило | Почему |
|---|---------|--------|
| 1 | **Один аккаунт — один IP** | Смешивание связывает цепочку аккаунтов |
| 2 | Не заходить в ферму с **домашнего IP** | Личный след + фермерский след = связь |
| 3 | Один IP на весь цикл аккаунта | Outlook → Steam → Guard → gift — с одного прокси |
| 4 | Не открывать два профиля на одном IP «на минутку» | Достаточно для связывания |
| 5 | План 10+ аккаунтов = бюджет на прокси **с первого дня** | Экономия здесь бьёт по всей ферме |

> Это не та статья расходов, где стоит экономить в ущерб безопасности.

## Выбор региона: Индия

На момент написания гайда **Prime дешевле на индийских аккаунтах** — примерно на **$0.92** меньше, чем на RU.

| Параметр | Индия | RU (для сравнения) |
|----------|-------|-------------------|
| Prime Status | Дешевле (~92¢ экономии) | Дороже |
| Валюта магазина | INR (рупии) | RUB |
| Gift-карты | Только индийские, в рупиях | Российские / другие регионы |
| Прокси | Индия, стабильный гео | По выбору |

На **100 профилях** разница только на Prime — уже **~$92** плюс каскад на gift и комиссии. На масштабе регион — часть юнит-экономики, а не косметика.

### Алгоритм выбора региона

1. Выбрали страну (рекомендация гайда: **Индия** под дешёвый Prime).
2. Включили VPN/прокси на **этот** регион.
3. Проверили IP: страна и город совпадают.
4. Только после этого — Outlook, Steam, Guard.

> Выбрали страну — VPN на нужный регион — можно регистрироваться.

## Бюджет и юнит-экономика

| Статья | Периодичность | Заметки |
|--------|---------------|---------|
| Прокси на 1 аккаунт | Ежемесячно | 1 IP = 1 аккаунт на весь жизненный цикл |
| Прокси на 50 аккаунтов | Ежемесячно | 50 отдельных IP (не «ротация на всех») |
| Happ Proxy / утилиты | Разово / по тарифу | Время на настройку заложить в процесс |

**Формула для планирования:**

```
Расход на прокси в месяц ÷ количество активных аккаунтов = прокси на 1 аккаунт
```

Вычитайте эту цифру **до** подсчёта прибыли с дропа. Прокси — операционный расход каждого аккаунта, как Prime.

> Прокси — **ежемесячная подписка**. Заложите её в юнит-экономику каждого аккаунта.

## Чек-лист перед регистрацией

- [ ] Оплачен личный/выделенный прокси, не бесплатный VPN
- [ ] Happ Proxy (или аналог) установлен и подключается
- [ ] IP проверен: страна и город = выбранный регион (Индия)
- [ ] Данные прокси сохранены в таблице: `host:port:login:password`
- [ ] Понятно, что этот же прокси пойдёт в AdsPower и NebulaAuth
- [ ] Домашний IP не используется для фермерских действий
- [ ] Бюджет на N аккаунтов = N прокси на месяц

## Частые ошибки

| Ошибка | Последствие |
|--------|-------------|
| Бесплатный VPN «на попробовать» | Грязный IP, капча, связанные аккаунты |
| IP показывает Европу вместо Индии | Outlook/Steam запоминают несостыковку |
| Один прокси на два аккаунта | Цепочный бан, проблемы с Guard |
| Guard привязали с домашнего Wi-Fi | Подтверждения по почте, урезание трейдов |
| Письмо Steam открыли без прокси | Новый след, рассинхрон с регистрацией |
| Экономия: отключили прокси «на час» | Случайный вход с реального IP |
| Не проверили прокси в AdsPower перед Steam | Час отладки и переделки профиля |
| Забыли про ежемесячную подписку | Протухший IP — потеря доступа к цепочке |
