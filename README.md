## О чём этот гайд

Этот гайд закрывает тему фермы кейсов в CS2 **от начала до конца** — без разрозненных советов из форумов. Здесь связная система, где **каждый шаг опирается на предыдущий**.

Каждый раздел — практический шаг, который вы повторяете у себя на экране. Сначала объясняется **зачем** это нужно, затем **как** сделать, чтобы вы понимали логику, а не просто копировали чек-лист.

### Как пользоваться

1. Идите **по порядку** — каждый раздел зависит от предыдущего
2. На каждом этапе сначала **один аккаунт вручную**, потом пачками по 10–15 в день
3. Ведите **таблицу аккаунтов** с первого дня ([Notion](chapters/32-notion-table.md) или шаблон в «Операционка»)
4. Начните с **3 аккаунтов**, пройдите весь цикл до вывода одного кейса через буфер, затем масштабируйтесь

---

> ⚠️ **Важно:** не смешивайте личную игру и ферму на одном железе без изоляции. Основной Steam — вне песочницы, на домашнем IP. Ферма — отдельная вселенная.

### Базовая гигиена (не меняется от патча к патчу)

- Один аккаунт — один IP
- Steam Guard привязывается с **того же IP**, что и регистрация
- Дроп не льётся напрямую на основной аккаунт
- Gift-карты строго под регион аккаунта
- Буферные аккаунты между фермой и хранилкой
